# nChain BSV Groth16 verifier (BLS12-381) — provenance

The real, on-chain nChain Groth16 verifier, pulled from BSV mainnet. Generated by
`nchain-innovation/zkscript_package`; deployed July 2024.

## Transactions (WhatsOnChain, BSV mainnet)

- **Verifier (locking script)** — parent tx
  `79a5281c9ea718c2a6be4e8d0e48b27855969d780c50393c01ad6267a9e24940`, output 0
  (~480 KB, the Groth16 verifier).
- **Proof (unlocking script)** — spending tx
  `e4cd00c1fa7dd6931dd1e45034e9d9f732e6d7d38f7826341715f488a146514c`, input 0
  (~40 KB, the proof + witness; spends the output above).

## Files

| file | what |
|------|------|
| `parent-tx.hex` | raw parent tx (contains the verifier locking script in vout[0]) |
| `spending-tx.hex` | raw spending tx (contains the proof unlocking script in vin[0]) |
| `parent-tx.meta.json` / `spending-tx.meta.json` | WhatsOnChain metadata (big scripts truncated by the API) |
| `groth16-locking.asm` | disassembled verifier opcodes (482,601 bytes, 476,052 opcodes, 50 distinct) |
| `groth16-unlocking.asm` | disassembled proof/witness (push-only) |

The `.hex` blobs are gitignored. Re-fetch them from WhatsOnChain and regenerate
the listings:

```
pnpm fetch:nchain      # downloads parent-tx.hex + spending-tx.hex
pnpm nchain:extract    # regenerates the groth16-*.asm opcode listings
```
